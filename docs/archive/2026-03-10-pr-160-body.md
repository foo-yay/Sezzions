## Summary
Implements Issue #160 by adding:
1) Site-level `playthrough_requirement` (default `1.0`) across schema/model/repo/service/facade/UI.
2) End Session toggle `Auto-Calculate End Redeemable SC` with real-time calculation and lock/unlock behavior.

## What changed
- `sites` schema now includes `playthrough_requirement REAL DEFAULT 1.0`.
- Added `DatabaseManager._migrate_sites_table()` and wired migration on startup.
- `Site` model now validates positive `playthrough_requirement`.
- Site repository/service/facade now persist and expose playthrough requirement.
- Setup → Sites table/add/edit/view dialogs now display/edit playthrough requirement.
- End Session dialog now includes auto-calc toggle:
  - Default OFF (preserves prior manual behavior)
  - ON: ending redeemable is read-only and auto-updates from start/end balances + site playthrough.
