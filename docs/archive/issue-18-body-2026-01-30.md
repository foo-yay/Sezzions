## Summary
Reset Database is broken due to regressions in ResetDialog.

Two observed failures:
- Reset button never enables (button gating logic regressed).
- Reset flow crashes when reading dialog options.

## Impact
- Tools -> Reset Database workflow is unusable.
- Blocks critical recovery / maintenance operations.

## Scope
- ResetDialog: ui/tools_dialogs.py
- Reset entrypoint: ui/tabs/tools_tab.py

## Steps to reproduce
1. Open the app
2. Tools tab -> Database section
3. Click Reset Database
4. Try full reset (disable Preserve setup data)
5. Attempt to satisfy the confirmations

## Expected
- Reset button enables only when checkbox is checked
  and confirmation text equals DELETE.
- Clicking Reset proceeds without exceptions.

## Actual
- AttributeError in ResetDialog button update path (missing _is_updating_size).
- AttributeError in Tools reset handler when calling dialog.should_preserve_setup().

## Severity
High

## Proposed fix
- Restore ResetDialog gating logic (no restore sizing logic).
- Restore should_preserve_setup() API.
- Add tests for the dialog enable rules and API contract.
