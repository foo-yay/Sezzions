## Summary

Refactor-only change: splits the monolithic `App.jsx` (3,176 lines) into a modular component architecture. No behavior changes.

**App.jsx: 3,176 lines -> 38 lines.**

## Architecture

```
web/src/
  App.jsx                          # 38 lines: route + auth + shell selection
  hooks/
    useAuth.js                     # Auth state, bootstrap orchestration
  services/
    api.js                         # authHeaders, getAccessToken, describeFetchFailure
    routing.js                     # Hash-based route utilities
  components/
    AppShell.jsx                   # Signed-in shell (topbar, rail, modals)
    MarketingShell.jsx             # Signed-out landing page
    MigrationShell.jsx             # /migration page with upload state
    common/
      Icon.jsx                     # SVG icon library
      ConfirmationModal.jsx        # Reusable confirm dialog
      StatusModal.jsx              # Connection health dialog

      AccountModal.jsx             # Account info + sign out
      SettingsModal.jsx            # Settings placeholder
      HighlightMatch.jsx           # Search text highlighting
    UsersTab/
      UsersTab.jsx                 # Self-contained users component (~800 lines)
      UserModal.jsx                # User CRUD modal
      ExportModal.jsx              # CSV export modal
      UserHeaderFilterMenu.jsx     # Column sort/filter dropdown
      FilterTreeNode.jsx           # Recursive checkbox tree node
      TableContextMenu.jsx         # Right-click context menu

      usersUtils.js                # Data manipulation utilities
```

## Key Design Decisions

- **UsersTab is self-contained**: manages its own state (users, selection, modals, confirmation, keyboard nav). Receives only `apiBaseUrl` and `hostedWorkspaceReady` as props.
- **useAuth hook**: encapsulates all auth/API/bootstrap orchestration. Returns auth state + actions.
- **Shell components unmount on sign-out**: when `sessionEmail` becomes null, `AppShell` unmounts naturally, resetting all shell/user state without explicit cleanup.
- **Escape key coordination**: UsersTab uses `event.stopImmediatePropagation()` to prevent AppShell's handler from firing when user modals are open.
- **Migration page optimization**: MigrationShell doesn't render UsersTab, so users API calls are no longer made on the migration page.

## Test Changes

## Pitfalls / Follow-ups

- **Tab switching resets users state**: switching away from the Users tab and back would remount UsersTab, losing loaded users. Not an issue today since no other tabs are enabled.
- **User loading timing**: users now load via a `useEffect` watching `hostedWorkspaceReady` instead of being called in parallel with import plan during bootstrap. The delay is imperceptible (one extra render cycle).
