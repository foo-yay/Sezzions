# Adjustments & Corrections (Issue #54)

## Summary

Implements manual adjustments system for correcting accounting issues:
- **Basis Corrections**: Delta adjustments to cost basis (e.g., missed fees, refunds)
- **Balance Checkpoints**: Known balance anchors at specific timestamps for reconciliation

## Implementation

### Data Layer
- New `account_adjustments` table with soft delete support
- `Adjustment` model with two types: `BASIS_USD_CORRECTION` and `BALANCE_CHECKPOINT_CORRECTION`
- `AdjustmentRepository` with full CRUD operations (12 unit tests)

### Service Layer
- `AdjustmentService` with validation and business logic (15 unit tests)
- **Checkpoint Integration**: `GameSessionService.compute_expected_balances()` uses latest checkpoint before cutoff
- **Basis Adjustment Integration**: `RecalculationService` injects basis adjustments as synthetic lots (negative IDs) in FIFO rebuild

### UI (Tools Tab)
- New Adjustments section with three dialogs:
  - **New Basis Adjustment**: user/site, date/time, delta amount, reason
  - **New Balance Checkpoint**: user/site, date/time, total SC, redeemable SC, reason
  - **View Adjustments**: table view with filters, soft delete, restore
