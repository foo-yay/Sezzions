## Summary

Closes #248. Extracts the shared EntityTable hook + component from duplicated tab code.

Three entity tabs (UsersTab, SitesTab, CardsTab) shared ~73% identical code across ~4,195 lines. This PR extracts the shared logic into a reusable hook and component, reducing each tab to a thin config layer.

## What Changed

### New shared infrastructure
- **`useEntityTable.js`** (~600 lines): Core hook encapsulating all common state and behavior -- data loading with pagination/fallback, filtering, sorting, keyboard navigation, copy-to-clipboard, context menu, column resize, infinite scroll, CRUD operations.
- **`EntityTable.jsx`** (~300 lines): Shared JSX component rendering the full table UI (toolbar, search, headers, data rows, summary rail, modals, context menu).
- **`tableUtils.js`** (~175 lines): Generic utility functions (highlight, normalization, column value getters, filter/suggestion builders).
- **Shared components** moved to `common/`: ExportModal, FilterTreeNode, TableHeaderFilterMenu, TableContextMenu.

### Tab migrations
| Tab | Before | After | Reduction |
|-----|--------|-------|-----------|
| UsersTab | ~1,420 lines | ~140 lines | 90% |
| SitesTab | ~1,379 lines | ~120 lines | 91% |
| CardsTab | ~1,396 lines | ~145 lines | 90% |

Each tab is now a thin config object + entity-specific render function + entity modal.

### Entity-specific customization points
- `extraToolbarButtons` -- per-entity action buttons (e.g., "Seed Demo Users" in dev mode)
- `extraLoaders` -- FK data loading (Cards loads Users for dropdown)
- `renderCell` / `getCellDisplayValue` -- custom cell rendering per entity
- `formToPayload` -- custom form serialization (Sites parses floats)
- `searchFilter` / `numericSortColumns` -- per-entity search/sort config

### Deleted files
- `UsersTab/ExportModal.jsx` (moved to common/)
- `UsersTab/FilterTreeNode.jsx` (moved to common/)
- `UsersTab/TableContextMenu.jsx` (moved to common/)
- `UsersTab/UserHeaderFilterMenu.jsx` (renamed to common/TableHeaderFilterMenu.jsx)

## Pitfalls / Follow-ups
- Future entity tabs (GameTypes, RedemptionMethods, etc.) can now be added as ~120-line config files
- The `useEntityTable` hook is large (~600 lines) -- could be split into sub-hooks if it grows further
- No frontend automated tests yet; consider adding React Testing Library tests for EntityTable
