## Summary
Fix spreadsheet selection stats/copy so hidden rows/columns (from filtering/search) are not included in the extracted selection grid.

Closes #50.

## Root cause
`SpreadsheetUXController._extract_table_selection()` treated all cells inside `selectedRanges()` as selected, even if the row/column was hidden. A shift-click range selection across visible rows could span hidden rows, inflating sums.

## Changes
- Ignore hidden rows/columns when extracting selection grids for:
  - QTableWidget
  - QTableView
- Add regression unit test covering hidden-row-in-range selection.

## Test plan
- `pytest -q`

## Pitfalls / follow-ups
- This fix is extractor-level (shared by many tabs). If any tab intentionally wants “include hidden rows” semantics in the future, it should opt in explicitly rather than relying on the shared defaults.
