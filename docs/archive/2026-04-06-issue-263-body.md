## Proposal

### Phase A: Table Column Parity (this issue - implement now)

1. Reorder columns to match desktop: Date/Time, User, Site, Cost Basis, Amount, Unbased, Type, Receipt, Method, Processed, Notes
2. Date column shows full datetime (date + time)
3. Add Unbased column (client-side: max(0, amount - cost_basis))
4. Add Processed column (checkmark display)
5. Remove Fees, Status, Net P&L columns from table (keep in modal only)
6. Receipt column shows "PENDING", "CANCELED", or "PENDING CANCEL" when no receipt_date, matching desktop

### Phase B: Row Color Coding (this issue - implement now)

1. Add status-based row coloring matching desktop palette
2. Red (#c0392b) for total loss (amount = $0)
3. Gray (#95a5a6) for CANCELED
4. Purple (#8e44ad) for PENDING_CANCEL
5. Orange (#e67e22) for PENDING with no receipt_date

### Phase C: Quick Actions (this issue - implement now where backend exists)

1. Cancel button (single selection, PENDING only, no receipt_date) - backend endpoint EXISTS
2. Uncancel button (single selection, CANCELED only) - backend endpoint EXISTS
3. Mark Received button (bulk, date picker dialog) - needs new bulk API endpoint
4. Mark Processed button (bulk toggle) - needs new bulk API endpoint

### Phase D: Bulk API Endpoints (this issue - implement now)

1. POST /v1/workspace/redemptions/bulk-mark-received (set receipt_date on multiple)
2. POST /v1/workspace/redemptions/bulk-mark-processed (set processed=true on multiple)
