## Summary

Implements Issue #97: Two-tier audit retention with meaningful summaries, configurable pruning, CSV export, and viewer improvements.

## Changes

### 1. Summary Data (Part 1)
- **Database**: Added `audit_log.summary_data` TEXT NULL column via migration
- **Service**: `AuditService.build_summary()` generates compact JSON summaries for purchases, redemptions, game_sessions
- **Integration**: All audit log methods (`log_create`, `log_update`, `log_delete`) now capture and store summaries
- **Tests**: 10 unit tests covering summary generation for all tables + edge cases

### 2. Retention Settings & Pruning (Part 2)
- **Service**: `AuditService.get/set_max_audit_log_rows()` for settings persistence
- **Service**: `AuditService.prune_audit_log()` atomically deletes oldest rows when limit exceeded
- **Default**: 10,000 rows (0 = unlimited)
- **Atomicity**: Pruning uses transaction with rollback on failure
- **Tests**: 9 unit tests covering retention settings, pruning logic, atomicity, unlimited mode

### 3. Settings UI (Part 3)
- **Dialog**: Added "Audit Log Retention" spinbox in Settings → Data section
- **UX**: Matches existing undo retention control pattern (0-100000, step 1000, "Unlimited" for 0)
- **Auto-prune**: Triggers pruning immediately when limit changes on save

### 4. CSV Export (Part 4)
- **Service**: `AuditService.export_audit_log_csv(output_path, start_date=None, end_date=None)`
- **Features**: Exports all columns including `summary_data`, optional date range filtering (inclusive)
- **Implementation**: Uses csv.DictWriter pattern, returns count of rows exported
- **Tests**: 5 unit tests covering export all rows, date filtering, empty results, summary inclusion

### 5. Viewer Improvements (Part 5)
- **Date Presets**: Added dropdown with Today, Last 7 Days, Last 30 Days, This Month, This Year, Custom, All Time
- **Sortable Columns**: Enabled table sorting by clicking column headers (ID, Timestamp, Action, Table, etc.)
- **Export Button**: Added "📊 Export to CSV" button that exports current filter results with date range filtering
- **Custom Dates**: Show/hide custom date inputs based on preset selection

### 6. Documentation (Part 6)
- **PROJECT_SPEC.md**: Updated audit retention section with summary data, retention settings, CSV export details
- **CHANGELOG.md**: Added comprehensive 2026-02-10 entry with problem/solution/implementation/testing details
