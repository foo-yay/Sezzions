# Schema/Spec Alignment: Add Missing Canonical Columns (Issue #136)

Closes #136

## Problem

Three tables were missing canonical columns in the `CREATE TABLE` statements
inside `DatabaseManager`, causing fresh databases to diverge from real
upgraded databases.  Appendix A in PROJECT_SPEC.md reflected the broken
fresh-DB schema, so the mismatch was invisible to `validate_schema.py`.

| Table | Missing columns |
|-------|----------------|
| `daily_sessions` | `notes TEXT` |
| `game_sessions` | `tax_withholding_rate_pct REAL`, `tax_withholding_is_custom INTEGER DEFAULT 0`, `tax_withholding_amount REAL` |
| `expenses` (migration rebuild) | `expense_entry_time_zone TEXT` — data-loss bug when `DEFAULT 'Other Expenses'` rebuild ran |

## Changes

### `repositories/database.py`
- **`daily_sessions` CREATE TABLE**: added `notes TEXT`.
- **`_migrate_daily_sessions_table()`**: converted from no-op to an ALTER TABLE migration that adds `notes` to existing databases.
- **`game_sessions` CREATE TABLE**: added all three `tax_withholding_*` columns.
- **`_migrate_game_sessions_table()` ALTER TABLE list**: added `tax_withholding_rate_pct`, `tax_withholding_is_custom`, `tax_withholding_amount` so existing databases are migrated.
- **`_migrate_expenses_table()` rebuild**: fixed `expenses_new` CREATE TABLE and `INSERT...SELECT` to include `expense_entry_time_zone`, preventing data loss.

### `docs/PROJECT_SPEC.md`
- Appendix A DDL updated: `daily_sessions`, `expenses`, and `game_sessions` sections now match the corrected fresh schema.

### `tests/integration/test_schema_alignment.py` (new)
11 tests total covering the torture-test matrix:

| Category | Tests |
|----------|-------|
| Happy path — fresh DB | `daily_sessions.notes`, `game_sessions.tax_withholding_*`, `expenses.expense_entry_time_zone` |
| Regression guards | `wager_amount`/`rtp` still present in `game_sessions`; core columns intact |
| Migration path | `_migrate_daily_sessions_table` adds `notes`; `_migrate_game_sessions_table` adds tax columns |
| Failure injection | `expenses_new` rebuild preserves `expense_entry_time_zone` column and data |

## Invariants Verified

- Only the three listed tables were touched.
- No existing accounting/FIFO logic changed.

## Pitfalls / Follow-ups

- The `game_sessions` migration table rebuild (triggered for old NOT NULL `game_id` DBs) also drops `wager_amount` and `rtp`. This is a separate, older bug not in issue #136 scope — proposed as a follow-up issue.
- `validate_schema.py` can now be run against a fresh `:memory:` DB to confirm no drift; this is not yet wired into CI.
