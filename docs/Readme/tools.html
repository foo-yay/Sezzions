Tools (deep dive)

Tools lives under Setup → Tools. It’s the “maintenance and correctness” area: recalculation, CSV import/export, adjustments, and database safety operations.

Read this before clicking destructive buttons. Backups and restores are safe only if you understand the mode you’re choosing.

Repair Mode

Repair Mode disables automatic rebuilds of derived data after edits. Instead, Sezzions records stale (User, Site) pairs that need rebuilding.

  • Status: shows whether Repair Mode is enabled
  • Stale pairs: number of pairs marked stale (with earliest boundary + reasons)
  • Rebuild stale pairs: runs rebuild(s) for those pairs
  • Clear stale list: clears the registry (use only if you intentionally want to ignore staleness)
In Repair Mode, tables can look incorrect. That’s expected until you rebuild.

Recalculation Tools

Recalculation rebuilds derived accounting data. In general: if you’re unsure about scope, use Recalculate Everything.

Recalculate Pair (scoped)

  1. Choose User and Site (autocomplete fields).
  2. Click Recalculate Pair.
  3. Wait for the progress dialog; review the result summary.
When to use: you edited data for one site/user and want quick rebuild without touching everything.

Recalculate Everything (global)

  1. Click Recalculate Everything.
  2. Wait for progress; this may take time on large databases.
  3. When done, sanity-check Unrealized/Realized/Daily Sessions.
When to use: after CSV imports, restores/merges, or after long periods in Repair Mode.

CSV Import / Export

CSV workflows are designed to be preview-first. Imports show what would be added/updated/conflicted before writing.

Import CSV

  1. Select the entity type to import (purchases, redemptions, sessions, etc.).
  2. Pick a CSV file.
  3. Review Import Preview tabs: To Add / To Update / Conflicts / Duplicates / Errors.
  4. Choose conflict behavior (skip vs overwrite) if conflicts exist.
  5. Proceed. After import, you’ll typically be prompted to run recalculation.
Rule: After importing transactional data, run recalculation so FIFO and reporting views are correct.

Export CSV

  • Most tabs have “Export CSV” for the currently filtered view.
  • Tools can export broader datasets depending on options.

Download templates

Tools can generate CSV templates (often as a ZIP) so you can fill them in and import.

Adjustments & Corrections

Use these to correct accounting without rewriting historical inputs.

  • New Basis Adjustment: adds a basis delta that participates in FIFO ordering
  • New Balance Checkpoint: establishes a known balance at a timestamp for expected-balance calculations
  • View Adjustments: review, soft-delete, restore, filter

Database Tools (Backup / Restore / Reset)

These operations acquire an exclusive tools lock so you can’t accidentally run concurrent destructive operations. Long operations run in a background worker and show a progress dialog.

Backup

  1. Set Backup Location.
  2. Click Backup Now.
  3. Verify the created file exists and is non-trivial in size.
Automation: Auto-backup can be enabled on a day interval (works only after backup location is set).

Restore

Restore supports multiple modes (exact names may vary in the dialog):

  • Replace: replaces the database with the backup. For safety, Sezzions creates a pre-restore safety backup first.
  • Merge all: merges records from backup into current DB.
  • Merge selected: merges only selected tables (useful for partial restores).
After restore/merge: run recalculation unless you are certain the backup’s derived tables are coherent for your current settings.

Reset

Reset deletes data. The dialog can optionally preserve setup/catalog tables. Reset will ask for confirmation, and can offer to create a safety backup first.

Reset cannot be undone. Prefer backups + restores unless you truly want to wipe.
