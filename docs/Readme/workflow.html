Daily workflow

This is a recommended “operator loop” for using Sezzions without accidentally drifting derived calculations. It assumes you’re tracking sweep coins (SC) and want FIFO basis, realized cashflow, and (optionally) taxable session P/L.

One-time setup

  1. Go to Setup and create at least 1 User + 1 Site.
  2. Optional but recommended: set up Cards, Redemption Methods, Game Types, and Games.
  3. Open Settings (gear icon) and pick a Theme + tax/withholding settings if you use them.

Each time you buy SC

  1. Open Purchases → Add Purchase.
  2. Enter date/time, amount (USD basis), SC received, and (if you track it) starting SC balance snapshot.
  3. Save. In normal mode Sezzions triggers a scoped rebuild for the affected (user, site) pair.
Tip: Starting SC balance snapshots are used in expected-balance checks. If you don’t track them, leave them consistent/blank per your workflow and rely more on recalculation + session checkpoints.

During play (optional, for taxable session P/L)

  1. Open Game Sessions → Start Session.
  2. Enter starting total SC and redeemable SC as shown on the site.
  3. When done, select the active session → End Session.
Session P/L is model-driven: It’s sensitive to accurate start/end balances and can intentionally differ from a simple “money in / money out” story session-by-session. Read Taxable session calculations before relying on the numbers.

Each time you cash out (redeem)

  1. Open Redemptions → Add Redemption.
  2. Enter amount and pick the redemption type:
    • Closeout: indicates no meaningful balance remains; closes the basis period.
    • Partial: indicates you are leaving meaningful balance; basis period continues.
  3. Save. Sezzions rebuilds FIFO allocations and updates realized cashflow (and can affect sessions).

End of day / weekly hygiene

  • Check Unrealized for open positions and sanity-check estimated SC.
  • Check Realized for realized cashflow and position notes.
  • Check Daily Sessions for tax rollups and notes.
  • Use Tools → Backup Now routinely.

When things look wrong

First question: Are you in Repair Mode? If yes, derived results will stay stale until rebuilt.
  1. Go to Tools → Repair Mode and confirm status.
  2. If not in Repair Mode: run Tools → Recalculate Pair for the affected user/site, or “Recalculate Everything” if unsure.
  3. If you imported data: run Recalculate Everything.
  4. If the app starts in Maintenance Mode: follow the Maintenance Mode guidance.
