Game Sessions tab

Game Sessions are used by the “taxable session P/L” model. A session is a time-bounded window with derived fields that are rebuilt during recalculation.

Common workflow

  1. Create a session with start/end timestamps.
  2. Enter any session notes/tags you want for reporting.
  3. Run recalculation if prompted (or from Tools) to rebuild derived session metrics.

Important notes

  • Sessions often depend on other events (purchases/redemptions/cashback) within their window.
  • Changing session timestamps can change which events are associated.
  • Derived session outputs are rebuilt, not hand-edited.
For the exact scope of session rebuilds and what is considered “derived”, see Recalculation & calculations.
