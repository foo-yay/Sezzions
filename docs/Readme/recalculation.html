Recalculation & calculations

Big idea: Sezzions has inputs (what you enter) and derived data (what it calculates). Recalculation rebuilds the derived data so the tables and reports match your authoritative inputs.

What recalculation does (scope)

The Tools recalculation pipeline is designed to match legacy behavior: it rebuilds FIFO + sessions together. At a high level it does:

  • FIFO rebuild: rebuilds purchase→redemption allocations and updates realized cashflow rows
  • Session recalculation: recalculates game session derived values for taxable P/L model
  • Cashback recalculation: recalculates cashback fields for the affected pair(s)
  • Event linking rebuild: rebuilds session/event link rows for navigation consistency
Important: recalculation can retroactively overwrite derived fields. If you need to preserve historical derived fields, that’s a separate feature decision.

Automatic rebuilds vs Tools rebuilds

In normal mode, most create/edit/delete operations trigger a scoped rebuild for the affected (User, Site) pair. This keeps the app feeling “live” without requiring you to constantly press Recalculate.

SituationWhat usually happensWhat you may need to do
Edit a purchase/redemption affecting accounting Scoped rebuild (pair) from an earliest boundary Nothing, unless in Repair Mode
Bulk CSV import Import writes authoritative rows; derived may be stale Run Tools → Recalculate Everything
Database restore/reset DB content changes massively Recalculate Everything after restore/merge (unless the restore already contains coherent derived tables)
Repair Mode enabled No auto rebuild; pairs marked stale Tools → Rebuild stale pairs / Recalculate pair(s)

FIFO basis allocation (Purchases → Redemptions)

FIFO means older purchases are consumed first when computing the cost basis of a redemption. Sezzions tracks this by writing allocation rows (a redemption can pull basis from many purchases).

Closeout vs partial redemption

  • Closeout (more_remaining = 0): treat redemption as “cash out everything”; can consume all remaining basis up to that point.
  • Partial (more_remaining = 1): consume only the redemption amount; basis period continues.
Why it matters: closeouts define “basis periods” used for balance-chain warnings and some position views.

Free SC redemptions

If a redemption is marked as Free SC, it typically has zero basis. That means:

  • Cost basis ≈ 0
  • Unbased portion ≈ full amount

Realized cashflow rows

After FIFO allocation, Sezzions can produce realized transaction rows representing cashflow P/L. This is the primary “realized” accounting output derived from redemptions.

Balance checkpoints & expected balances

Sezzions computes “expected balances” at timestamps to help detect data-entry mistakes. Tools can also create balance checkpoints that override prior expectations.

Operator takeaway: If expected-balance labels are warning you, you either (a) entered a wrong start balance, (b) missed a redemption/purchase, or (c) need to add a checkpoint to represent reality.

Basis adjustments

Basis adjustments are synthetic “purchase-like” entries that participate in FIFO ordering. They can increase or decrease basis without rewriting historical purchases.

Repair Mode and stale pairs

When Repair Mode is enabled, Sezzions will stop auto-rebuilding derived tables after edits. Instead it records which (User, Site) pairs are stale and the earliest timestamp boundary that needs rebuilding.

In Repair Mode, the app can look wrong on purpose. That is expected until you rebuild.

Taxable session calculations (Game Sessions)

Session P/L in Sezzions is a deliberate model:

  • Taxable results are recognized when a session is closed (ended), not when redeemable appears.
  • “Discoverable” redeemable is recognized at the start of a session if it exceeds the expected checkpoint.
  • Pending basis pools and locked/redeemable transitions can make per-session results diverge from intuition.
Change control: session model changes require scenario-based tests. Don’t tweak formulas casually.

Tax withholding bulk recalculation

Separately from FIFO/session rebuilds, Sezzions can bulk-recalculate tax withholding estimates for daily sessions. This is initiated from a dedicated dialog and can overwrite historical withholding values based on current Settings.
