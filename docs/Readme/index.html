What Sezzions is

Sezzions is a desktop app for tracking sweepstakes casino activity with accounting-grade outputs. You record purchases and redemptions, optionally record game sessions, and the app computes derived views like FIFO basis allocations, realized cashflow, taxable session P/L, and open/unrealized positions.

Authoritative inputs
  • Purchases (basis lots)
  • Redemptions (cashouts)
  • Game Sessions (for taxable P/L model)
  • Daily Sessions notes/rollups (for reporting)
  • Adjustments & checkpoints (Tools)
Derived / rebuildable data
  • FIFO allocations (purchase→redemption mapping)
  • Realized transactions (cashflow P/L rows)
  • Session-derived fields (taxable P/L components)
  • Event links (cross-navigation between sessions and events)

When to use Tools

Most of the time, Sezzions rebuilds derived data automatically after edits. Tools becomes important when you:

  • import CSVs in bulk and need to rebuild derived data afterward
  • restore/merge/reset databases
  • enable Repair Mode (advanced) and need to manually rebuild stale pairs
  • create basis corrections or balance checkpoints
Key safety rule: Backup before restore/reset. And read Tools (deep dive) before using destructive operations.

Quick start

python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python3 sezzions.py

Default DB file is ./sezzions.db (repo root). Override with SEZZIONS_DB_PATH.
