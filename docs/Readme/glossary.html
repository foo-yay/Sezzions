Glossary

TermMeaning (operator-focused)
Authoritative Records you enter directly (purchases, redemptions, sessions). These are the “source of truth”.
Derived Rebuildable outputs computed from authoritative inputs (FIFO allocations, realized rows, session-derived values).
FIFO First-In-First-Out basis allocation: older purchases are consumed first by redemptions.
Cost basis The purchase basis allocated to a redemption (derived). Used in realized reporting.
Unbased Redemption amount not covered by allocated basis (derived).
(User, Site) pair A scoped “account” for recalculation and staleness tracking.
Repair Mode Advanced mode that disables automatic rebuilds and records which pairs are stale until manually rebuilt.
Recalculate Pair Tools action that rebuilds derived data for one (User, Site) scope.
Recalculate Everything Tools action that rebuilds derived data for all users/sites.
Checkpoint A recorded “known balance” at a timestamp used for expected-balance tracking (correction mechanism).
Adjustment A correction record (e.g., basis adjustment) that participates in rebuild logic without rewriting history.
