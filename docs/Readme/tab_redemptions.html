Redemptions tab

Redemptions are the authoritative input for cashouts/withdrawals. Redemptions drive FIFO consumption of purchases.

Common workflow

  1. Filter to a specific User + Site.
  2. Add each redemption with accurate timestamp and amount.
  3. Use View Redemption to inspect allocations and related data.

Key derived columns

  • Cost Basis: FIFO-allocated basis from purchases (derived).
  • Unbased: portion of redemption amount not covered by basis (derived; clamped at 0).
These fields are rebuildable and come from the same underlying allocation logic as the View Redemption → Related tab.

What changes require rebuild

  • Editing redemption timestamp or amount can change FIFO allocations.
  • Editing purchases also changes redemption allocations.
If Repair Mode is enabled, redemptions may show stale basis/unbased until you rebuild from Tools.
