Common dialogs

This page lists the “high impact” dialogs where a wrong click can lead to confusing outputs (usually because derived data becomes stale until rebuilt).

Maintenance Mode

Maintenance Mode appears when Sezzions detects a database integrity / schema mismatch or when an operation requires temporarily restricting actions.

  • Read the message carefully; it typically states what action is required.
  • If prompted, follow through with Tools actions (backup, rebuild, restore, etc.).

Repair Mode confirmation / info

Repair Mode dialogs explain that auto rebuilds are disabled and you are responsible for rebuilding stale pairs.

Expect “incorrect-looking” derived views until you rebuild.

View Redemption

Use this dialog to inspect what a redemption is “made of”: which purchases were consumed (FIFO) and what cost basis was allocated.

CSV import preview

Imports are preview-first. Pay attention to errors and conflicts before you commit.

  • Errors: rows that cannot be imported; fix the CSV before proceeding.
  • Conflicts: existing rows that differ; choose skip vs overwrite deliberately.

Tax withholding recalculation

If you track withholding separately, this dialog can recompute withholding amounts over a range. Treat it like a targeted recalculation: run it deliberately, then review reports.
