Purchases tab

Purchases are the authoritative input for basis lots. Purchases feed FIFO allocation. If you edit purchases, you typically need a rebuild (automatic unless Repair Mode is enabled).

Common workflow

  1. Set filters (User, Site, date range) to focus on a single account.
  2. Add purchases as you buy currency/credits.
  3. Use View/Edit to correct dates, amounts, tags, and notes.

How it affects calculations

  • FIFO basis: purchases are consumed by redemptions in chronological order (see Recalculation).
  • Unrealized: remaining lots contribute to open/unrealized basis.
  • Realized: once allocated to a redemption, the lot’s basis becomes realized cost basis.
Repair Mode: edits mark the (User, Site) pair stale instead of rebuilding. Rebuild from Tools → Repair Mode.

Tips

  • Keep purchase timestamps accurate; FIFO depends on ordering.
  • If you bulk-import purchases, run recalculation afterward.
  • Use adjustments/checkpoints for corrections when appropriate (see Tools → Adjustments).
